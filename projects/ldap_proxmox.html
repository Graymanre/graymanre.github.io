Deploying LDAP on Proxmox

GraymanRe in Projects

Introduction

In a previous blog post, we dove into setting up a Windows domain and enable LDAP authentication throughout the domain. This allowed us to setup LDAP authentication against pfSense and ultimately, we setup an OpenVPN server on the pfSense which we could use to authenticate ourselves over LDAP against our domain for remote access to our Domain.
In this new blog post, I am diving into revamping the current setup to enable LDAPS and subsequently enabling LDAPS authentication against a Proxmox server. If you are interesting in setting up your own network as a Windows Domain, follow the previous steps in my blog post to create the following network layout:

  • 1 VM running Windows Server 2022, with Active Directory Domain Services role.
  • 1 VM running Windows Server 2022, with Active Directory Certificate Services role.
  • 1 pfSense appliance (version 24.03).
  • 1 (mobile) device, such as a laptop with the OpenVPN client installed.

We are adding one additional node to our infrastructure:
  • 1 device, such as an Intel NUC, full-fledged server or laptop running Proxmox VE.

Reconfiguring LDAP to support LDAPS

I don't need to explain for those familiar with LDAP authentication that LDAP is quite a weak protocol as it performs clear-text authentication over the network. This would allow for host of attack vectors and subsequently can lead to full domain compromise and takeovers. With LDAPS, the way of communication changes in one major aspect: although the protocol itself stays quite similar, a SSL/TLS is now added as a wrapper around LDAP, which encrypts the packages flowing through the network. This subsequently hampers LDAP injection and other types of clear-text LDAP attacks. It is by no means the best authentication method
